# 知识库 FAQ

> 说明：以下内容仅用于高置信度的自动回复场景。若问题超出标准路径、涉及个案判断或需要人工核实，请不要自动解决。

## 1. 发票下载 / 补发

### 问题
如何下载发票，或申请补发发票副本？

### 适用条件
- 用户已完成下单或付款。
- 发票信息与订单信息能够在系统中对应。
- 该问题只涉及标准的发票下载或补发流程。

### 标准答复
请先登录账户，进入订单详情、账单页或发票相关入口，查看是否已有可下载的发票文件。  
如果系统中已经生成发票，一般可以直接下载或重新下载。  
如果当前没有看到发票入口，请先确认订单是否已完成支付，随后再在订单相关页面查找发票申请或补发入口。  
如系统提供补发功能，请按页面提示提交即可。

### 不能自动解决的边界
- 订单与发票信息不匹配。
- 发票抬头、税号、金额等内容需要修改。
- 用户要求开具特殊类型发票。
- 系统中没有发票入口，或页面显示异常。
- 需要人工核实订单状态、付款状态或开票资格。

## 2. 修改密码 / 忘记密码

### 问题
如何修改密码，或者忘记密码后怎么找回？

### 适用条件
- 用户仍能访问自己的账户登录入口。
- 用户需要走标准的修改密码或重置密码流程。
- 不涉及账号被盗、邮箱失效、手机号失效等特殊情况。

### 标准答复
如果你记得当前密码，可以先登录账户，然后在账户设置、安全设置或个人资料页面中找到修改密码入口，按页面提示完成操作。  
如果你忘记了密码，请在登录页选择“忘记密码”或“重置密码”，再按提示通过已绑定的邮箱或手机号完成验证并设置新密码。  
完成后，请使用新密码重新登录。

### 不能自动解决的边界
- 用户无法访问绑定邮箱或手机号。
- 账号疑似被盗、被锁定或存在安全风险。
- 登录页面没有重置密码入口。
- 用户要求人工代为修改密码。
- 需要核验身份的特殊重置请求。

## 3. 课程退款标准流程

### 问题
如何申请课程退款？

### 适用条件
- 用户提出的是标准退款申请。
- 退款入口在系统中可见。
- 不涉及课程退款例外、特殊审批或争议处理。

### 标准答复
请先登录账户，进入订单详情、课程详情或帮助中心中的退款入口，按页面提示提交退款申请。  
通常需要确认对应订单或课程，并填写退款原因后提交。  
提交后，请按照系统页面的下一步提示继续操作。  
如果页面提供退款状态查询，也可以在订单或帮助中心中查看进度。

### 不能自动解决的边界
- 用户要求退款但当前没有可见退款入口。
- 需要人工审批、例外处理或额外资格判断。
- 涉及已超出标准退款流程的个案。
- 用户对退款结果有争议或需要人工复核。
- 需要确认政策细则、时效、扣费规则或特殊限制。

## 4. 退款进度查询

### 问题
退款申请提交后，如何查询退款进度或确认是否到账？

### 适用条件
- 用户已经提交过退款申请。
- 用户询问的是标准退款进度、处理状态或到账确认。
- 用户可以访问订单、课程或帮助中心中的退款记录入口。

### 标准答复
请先登录账户，进入订单详情、课程详情或帮助中心中的退款记录入口，查看当前退款状态。  
如果状态显示为“处理中”或类似提示，说明退款仍在系统处理流程中，请继续以页面显示为准。  
如果状态显示为“已完成”或类似提示，请再核对原支付渠道、银行卡或第三方支付账户的入账记录。  
如果页面提供退款编号或处理记录，请保留相关信息，便于后续需要人工核实时使用。

### 不能自动解决的边界
- 页面没有退款记录或状态查询入口。
- 用户称状态已完成但实际未到账。
- 退款状态长时间没有变化，需要人工核实。
- 用户要求确认具体到账时间、银行处理时效或支付渠道内部状态。
- 涉及多笔订单、部分退款、争议退款或异常扣费。

## 5. 撤销退款申请

### 问题
已经提交的退款申请可以撤销吗？

### 适用条件
- 用户只是询问是否可以撤销退款申请。
- 系统页面中存在退款记录、退款详情或撤销入口。
- 退款仍处于可操作的标准处理阶段。

### 标准答复
请先登录账户，进入订单详情、课程详情或帮助中心中的退款记录入口，查看该退款申请是否提供“撤销”“取消申请”或类似操作。  
如果页面提供撤销入口，请按页面提示确认撤销。  
如果退款已经进入处理、审核通过、款项退回或系统提示不可撤销，则通常无法通过标准入口撤销，请以页面提示为准。

### 不能自动解决的边界
- 页面没有撤销或取消申请入口。
- 退款已完成、已退回、已进入支付渠道处理或显示不可撤销。
- 用户撤销退款后仍希望恢复课程、订单或权益，需要人工核实。
- 用户要求客服代为撤销退款申请。
- 涉及审批、争议、异常扣费或其他非标准退款流程。

## 6. 退款失败处理

### 问题
退款失败、退回失败或退款被拒后应该怎么办？

### 适用条件
- 用户看到系统明确提示退款失败、退回失败或类似状态。
- 用户询问的是标准自助处理方式。
- 页面中可能提供重新提交、补充信息或更换退款方式的入口。

### 标准答复
请先登录账户，进入订单详情、课程详情或帮助中心中的退款记录入口，查看失败原因和页面提示。  
如果页面要求补充信息、重新提交申请或更换退款方式，请按提示完成操作。  
如果失败原因与原支付渠道、银行卡、账户信息或第三方支付账户有关，请先确认相关账户状态是否正常，再根据页面提示继续处理。  
如页面没有提供可继续操作的入口，请保留退款记录、订单信息和失败提示，联系人工客服核实。

### 不能自动解决的边界
- 页面未展示明确失败原因。
- 用户无法按页面提示重新提交或补充信息。
- 涉及银行卡、支付渠道、账户冻结、风控或身份核验。
- 退款被拒、金额不一致或用户对处理结果有争议。
- 需要人工查看订单、支付流水或退款记录。

## 7. 更改账户邮箱标准入口

### 问题
如何更改账户邮箱？

### 适用条件
- 用户可以正常登录当前账户。
- 账户设置中存在标准邮箱修改入口。
- 不涉及邮箱失效、账号安全事件或人工核验。

### 标准答复
请先登录账户，然后进入账户设置、安全设置或个人资料页面，查找邮箱管理或邮箱修改入口。  
按页面提示提交新的邮箱地址，并完成系统要求的验证流程。  
如果修改成功，系统通常会提示更新完成，并可能要求你使用新邮箱重新确认账户信息。

### 不能自动解决的边界
- 用户无法登录当前账户。
- 绑定邮箱不可用，导致无法完成验证。
- 页面中没有邮箱修改入口。
- 需要人工审核或安全核验后才能更改邮箱。
- 涉及账号接管风险、异常登录或其他安全问题。
